Background

VLM/VLA Serving 处于 "pre-vLLM" 阶段

LLM Serving — 已解决

  • vLLM / TRT-LLM / SGLang — 成熟的 KV-cache, paged attention, continuous batching
  • Prefill/Decode 分离已标准化
  • Benchmark 完善 (Chatbot Arena, LMSYS)

VLM/VLA Serving — 空白

  • 无统一 serving system — 每个模型自带推理脚本
  • Phase 定义不统一 — VLM (E/P/D) vs VLA (E/C/A) vs WAM (E/V/A)
  • 无标准 benchmark — 延迟数字不可跨论文比较
  • 瓶颈结构未知 — 不知道优化哪里
张昊路线: Parameter Server → Alpa → vLLM → FastVideo → VLM/VLA real-time systems
每一步都是 ML Systems 前沿的下一个自然问题。Measure first — 不凭直觉优化。

Beyond

三大范式之外 — 新兴架构

VLM / Flow-VLA / WAM 之外，4 个值得关注的新方向

Dual-System Architecture

GR00T N1 (NVIDIA), StreamVLA
VLM (慢, ~200ms, 低频 reasoning) + DiT (快, ~20ms, 高频 control)。
分频协作: slow path 出目标, fast path 出动作。

延迟不是单一数字 — 是 heterogeneous pipeline

MoE / MoT VLA

HY-Embodied-0.5 (Tencent+NVIDIA, 2B active)
Mixture-of-Transformers: sparse activation → edge-friendly。
2B active params 可能 ~30-60ms，开源可用。

MoE 对 inference latency 的影响是 open question

SSM / Linear Attention VLA

AnoleVLA (Mamba/RWKV backbone)
O(n) decode 替代 O(n^2) — 理论上 sequence length 无关。
3x faster than Transformer VLA baseline。

Architecture paradigm shift — 待验证实际延迟

1-Step Distilled Flow

FASTER, Mean-Flow VLA
把 multi-step Flow VLA 压缩到单步: 8.7-83.9x 加速。
Action-to-Action Flow: 0.56ms (理论下限)。

Flow → VA 速度，保留 Flow 泛化 — 最有前景的加速路线
共同趋势: 所有新范式都在打破 "延迟 vs 泛化" 的 Pareto frontier。
Serving system 层面的优化 (张昊实验室的强项) 是 所有范式都需要的 — 无论哪个赢。
Insight 1

Phase Bottleneck 因范式完全不同

Encode-Dominated (VLM / Flow-VLA)

  • Qwen2.5-VL-7B: Encode 58% — ViT 处理 5000+ image tokens
  • LingBot-VLA-4B: Encode ≈ Context 48% / 51% — ViT 和 KV fill 几乎等价
  • 优化方向: visual token pruning, EPD disaggregation
  • 文献支撑: FastV (95% pruning), ID-Selection (97.2%)

Action-Dominated (VA / WAM)

  • NitroGen: Action 91.6% — 12-layer DiT × k steps
  • Fast-WAM: Action 89% — 30-layer MoT × 10 steps
  • LingBot-VA: Action 68% + Video 28% — 5B DiT shared
  • 优化方向: step distillation, DiT caching, architecture search
两种完全不同的系统优化策略
VLM/VLA → token-level 优化 (pruning, scheduling)   |   VA/WAM → step-level 优化 (distillation, caching)

DiT 线性 & 超线性

  • NitroGen 174M: 7.2ms/step, k=1..16 完美线性 — compute-bound
  • 174M → 350M: 2x params, 4.4x latency — 超线性
  • 转换点在 174M-350M: compute-bound → memory-BW-bound
  • 5B WanDiT: 28.5ms/step — BW-bound regime 下 layer 数主导

Step Distillation = 实时化杠杆

  • NitroGen k=16: 126ms (7.9Hz)
  • NitroGen k=1: 17.9ms (55.9Hz)
  • 文献: consistency distillation 可达 1-step
  • k 是最强的延迟-质量旋钮

做对了什么

系统性偏差

  • Backbone scaling 用 param ratio 而非 FLOPs ratio
  • MoT cross-attn per-step cost 被严重低估
  • VLM→VLA attention 假设完全错误
Methodology alignment

Hao 的 Co-Design 方法论 — 我们在哪一层？

Layer Research activity Co-design?
L0 读论文 / survey No
L1 Profile 现有模型 — latency, attention 分布, per-phase breakdown No
L2 发现结构性矛盾 —"X 假设 vs Y 现实不匹配" 前提

我们当前位置

exp01-06 全部在 L1。exp05a 的 Gini 崩塌部分触及 L2，但未导向解法。

L1 积累是 Hao-style 工作的必要前提 — 没有 "60-80% KV 浪费" 就没有 vLLM。但停在 L1 永远做不到 co-design。

下一步 — L1 → L2 跨越

exp07 的任务不是"再 profile 一个模型"，而是把现有 L1 素材 缝合成一个 L2 结构性论证，然后 motivation 指向 L5 方向。

候选 D：EPDA 四阶段干扰量化 — 同时延伸 DistServe + FastVideo 两条主线。

见 Hao 的 one-liner: 不 pitch L1 的独立价值，pitch 对 L2→L5 的判断力。
