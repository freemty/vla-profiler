01 — Motivation

Why This Matters

张昊的技术路线: Parameter Server → Alpa → vLLM → FastVideo → VLM/VLA real-time systems. 每一步都是 ML Systems 前沿的下一个自然问题。

🞔
Robotics Control Loop
VLA needs to output actions at 10-100ms for real-time control. Current 7B VLA takes 200-500ms per action — too slow for dynamic tasks.
Target: <50ms
🚗
Autonomous Driving
End-to-end VLM perception-decision pipeline must run in real-time. Vision encoding alone takes 253ms on a single image.
Encode = 58% of total
🌐
No Unified Serving System
LLM has vLLM. VLM serving is fragmented — no EPD disaggregation, no modality-aware scheduling, no visual KV-cache optimization.
Pre-vLLM era
⚡
Algorithm-System Gap
Token pruning (97.2% reduction possible), quantization, and speculative decoding exist in isolation — no system co-designs them.
Biggest blank spot
